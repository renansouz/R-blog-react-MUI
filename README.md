## Introduction

R-Blog is a React blog interface built with TypeScript and Material UI. The project focuses on responsive article layouts, reusable UI components, theme management, and modern frontend structure. It can be used as a starter template for blog platforms, content-driven applications, developer portfolios, or editorial interfaces requiring dark mode support and Material UI integration.

The application is optimized for users searching for a React blog template, Material UI blog interface, TypeScript blog frontend, or responsive React article layout.

## Technical Overview

The project uses React with TypeScript for component development and static typing. Material UI provides the design system, layout primitives, typography, and theming infrastructure. The interface is structured around reusable components to simplify future extension and maintenance.

Theme switching is handled through centralized state and Material UI theme providers, allowing the application to toggle between light and dark modes consistently across all components. The project uses Vite for frontend tooling and development server performance, with pnpm for dependency management and package installation.

The architecture is suitable for extending into a full blog CMS frontend, Markdown renderer, or API-driven content platform.

## User Instructions

### Installation

```bash
git clone https://github.com/renansouz/R-blog-react-MUI.git
cd R-blog-react-MUI
pnpm install
```

### Run locally

```bash
pnpm dev
```

Open `http://localhost:5173/` in your browser.

## Developer Instructions

### Development server

```bash
pnpm dev
```

### Production build

```bash
pnpm build
```

### Preview production build

```bash
pnpm preview
```

### Linting

```bash
pnpm lint
```

### Contributing

- Create a feature branch from `main`.
- Keep pull requests focused on a single change.
- Use descriptive commit messages.
- Maintain component reusability and consistent naming patterns.
- Verify the project builds successfully before opening a PR.
- Include screenshots for visual or layout-related updates.

Suggested branch names:

```bash
feature/theme-refactor
fix/mobile-layout
chore/readme-update
```
